# PRD: 數位食譜管理平台

## 1. 專案概述
- **背景與動機**：現代人越來越重視健康與自己動手做料理，然而許多私房菜、家傳食譜或是網路上找來的健身餐單，往往散落於不同的筆記本、手機截圖或是社群平台收藏中。為了方便管理並延續料理智慧，我們需要一個專注於紀錄與管理的數位食譜平台。
- **目標用戶**：
  - 喜愛下廚的家庭主婦/夫
  - 希望記錄健身餐單的運動員
  - 想要數位化保存家傳食譜的料理初學者
- **核心價值主張**：提供一個操作直覺、易於分類與檢索的專屬食譜資料庫，讓每一次的料理經驗都能被完整記錄並輕鬆重現。

## 2. 功能需求
- **新增食譜**：
  - 作為使用者，我希望可以快速建立一篇食譜，以便把我的料理做法記錄下來。
- **材料與步驟清單**：
  - 作為使用者，我希望可以分條列出食譜所需的材料與料理步驟，以便在廚房料理時能一步步對照操作。
- **食譜分類標籤**：
  - 作為使用者，我希望可以為食譜加上自訂標籤（例如：中式、減脂、甜點），以便日後快速搜尋及過濾特定種類的食譜。
- **自動生成購物清單**：
  - 作為使用者，我希望系統能將選定食譜的材料匯總成購物清單，以便我去超市時不會忘記買任何食材。
- **評價與筆記功能**：
  - 作為使用者，我希望可以在每次實作食譜後留下評價與筆記，以便作為日後改進或調整口味的參考。

## 3. 非功能需求
- **技術限制**：使用 Flask + Jinja2 + SQLite 進行開發。
- **效能與安全考量**：
  - 系統應確保使用者在新增食譜時，圖片與文字內容上傳的順暢度（可考慮圖片壓縮）。
  - 需要基本的防禦機制（如防止 SQL Injection、XSS）來保護使用者的資料安全。
  - 網頁需具備基礎 RWD（響應式網頁設計），以便使用者在手機或平板上瀏覽食譜。

## 4. MVP 範圍

| 優先級 | 功能項目 | 說明 |
| --- | --- | --- |
| Must Have | 新增/編輯/刪除/瀏覽食譜 | 系統最基本的核心 CRUD 功能 |
| Must Have | 材料與步驟清單管理 | 食譜內容的關鍵資訊記錄 |
| Should Have | 食譜分類標籤與檢索 | 提升檢索體驗，方便管理食譜 |
| Should Have | 評價與筆記功能 | 增添食譜實作後的經驗傳承 |
| Nice to Have | 自動生成購物清單 | 進階便利功能，初期若時程緊迫可後續迭代 |
| Nice to Have | 食譜圖片上傳 | 更豐富的食譜展示（初期可用網址或不強制） |
